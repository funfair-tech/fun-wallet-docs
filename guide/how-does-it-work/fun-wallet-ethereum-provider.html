# What is an Ethereum provider ?

üí° Most dApps use either web3 or etherjs to read from and write to the blockchain directly from the client.

üí° These libraries follow best practices, and expose interfaces that should allow you to do everything necessary for blockchain development.

üí° Also, the nodes and signing all need to happen in the Wallet client not the dApp client as only the Wallet client has access to the necessary context information.

üí° An Ethereum provider is something you can inject into the constructor of web3 or etherjs and override the point at which it makes the JSON-RPC call.

üí° Minimal changes, if any, should be required to the web3/etherjs code.

üí° Once done, our Ethereum provider will proxy all the JSON-RPC calls to the Wallet.
